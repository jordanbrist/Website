3D Rigging and Animation: Wrap-Up

12-05-2023

I read some people's opinions online regarding using Linkedin Learning courses to learn new things and I found a pretty great analogy that I thought I should share. I am paraphrasing a bit due to the comment being poorly structured but it was basically this: "LinkedIn Learning isn't great for everything and many courses can be extremely un-rewarding or even unsatisfactory for learning a subject. When in-depth knowledge is needed, LinkedIn Learning has the attention span of a TikTok; it is so short and high level that someone new to the subject might get something out of it but it won't prep you for real life scenarios."

I think that I would have to agree with this as a whole. However, as someone that gets easily distracted, these shorter bites of content feel easier to digest and also allow me to take more time on the individual tools that I would normally gloss over. Then, after each topic, I sit and write down what I learned so that I can better retain that information for later use. Basically, I can take each of these less than 10 minute videos and turn it into an experience that spans almost 30 minutes or longer.





Section 12 - 6: Using Quick Rig

12-04-2023

Quick Rig is a simple tool that allows you to create a skinned and controllable rig in a super short amount of time. It still has some areas that need to be refined but the whole process takes less time than even automated programs built purely for this task like AccuRig or Mixamo. I am blown away by this tool and yet again see another reason to make the switch to Maya for rigging and animation of characters.

There are many different attributes that can be modified to create a specific look. This seems like a great and simple way to make stylized hair for a character model. It only takes a few clicks to set up a single strand and can be repeated over the entire scalp.

Section 12 - 1: Spring IK Solvers

12-04-2023

By enabling the Sprink Solver IK attribute, you can access a different way of moving object chains with IK. The green joints are using a standard IK solver and they lift off the ground and shrink in a way that may not always be right for every situation. With the Spring Solver, the blue joints move in a more even and predicatable manner, like a spring. This seems like it will be very useful for more mechanical purposes where a standard IK is more fitting for organic movement.

Section 11 - 5: Solidify Deformer

12-01-2023

The Solidify Deformer keeps geometry from being stretched or deformed. In this example, this character has a belt buckle that is part of the character's geometry. Normally this would make it very difficult to change the way it behaves when the rest of the model is stretching. However, by creating a component tag encapsulating the vertices of the belt buckle and adding the Solidify Deformer onto it, now it no longer stretches with the rest of the model. It retains its shape and moves with the surface. This seems like a very useful tool that I could see myself using a lot.





Section 11 - 4: Morph Deformer

12-01-2023

The Morph Deformer allows you to morph the geometry of a shape with another (kind of like blend spaces but for different use cases). In this exercise, we took a plain face and a duplicate of that geometry that had been altered and used the Morph Deformer to blend the geometry. The cool thing about this tool is that you can keep the original object intact while changing the features of the other object. This could be a good way to go about changing a character's emotion on the fly without too much trouble. I may have to play with this some more...





Section 11 - 3: Component Tags

12-01-2023

Using Component tags, you can take selections of vertices, edges, or faces and create groups called components. Then, in the attribute editor, you can select the component and it will select the group of vertices/faces/edges to be edited as if they were selected using the selection tool. This seems like a very useful tool and would save a lot of time in the modeling stage when working on a multi-faceted object with many intricate details.





Section 11 - 2: Move Skinned Joint Tool

12-02-2023

Yet another very quick demonstration. This time on the Move Skinned Joint Tool. This tool allows for the movement of a joint in an already skinned rig. Normally, if you were to move a joint in a skinned model, the skin will move with it and it will just deform the model. However, with this tool active, skinning is disabled and joints can be moved freely. Then, when you are finished with the tool, skinning is re-enabled and everything is back to normal with the exception of the new joint placement having taken effect.

Section 9 - 4: Creating a Head Aim Control

11-30-2023

In this exercise, we learned how to create an aim control for an object (a character's head) by using an aim constraint. However, when using an aim constraint, you can no longer aim the head at something other than the aim controller. Section 8 - 1: Using Nodes Instead of Constraints

11-29-2023

This was just a very quick demonstration of how one can use the node editor in place of constraints when parenting a joint to a control (in this instance, a circle control). I'm surprised that this is such a simple task and even more surprised that it wasn't shown earlier in the course and then used for things like ribbon rigging. Rather than parenting using a parent constraint, all you have to do is open the node editor with both items selected and drag the rotation and scale nodes from the parent to the child object. For translation, the only extra step that needs to be taken is to go into the object's attribute editor and copy the translation values to the Transform Offset Parent Matrix before resetting the transform attributes to zero and linking the nodes in the node editor. I don't know too much about the work that happens behind-the-scenes with parent constraints, but I do know that earlier in this course we learned that it was better to use Offset Parent Matrices and the Node Editor. Section 7 - 4: Controlling Stretch with Tension

11-27-2023

The Tension deformer is similar to Delta Mush in that it is used to smooth the distortion of vertices. However, Delta Mush is mainly used when squashing an object (like with the creasing of the leg and pelvis in the previous exercise), whereas Tension is best utilized in the stretching of a mesh. So, with this outstretched arm, the geometry starts to buldge a bit and become distorted as the shoulder is rotated. With Tension, that evens out the distance between the vertices and smooths out the whole thing, making it look much better. However, this also then caused there to be some distortion on the opposite side of the shoulder where vertices were bunched close together. Rather than painting the area to keep it from deforming, all that was needed to fix the problem was to reduce the Squash Constraint from 1 to 0. This will allow that shoulder to utilize the tension deformation when it stretches in the oposite direction as well.





Section 7 - 3: Smoothing Out Creases With Delta Mush

11-27-2023

Delta Mush is a deformer that is used to, in this scenario, smooth out creases in a mesh without needing to use weight painting (which most people consider to be a long and annoying process). While it can be applied to the whole mesh, it is better to use it on an isolated area. In this exercise, we used the used the painting tool to turn the Delta Mush that was applied to the entire mesh and reduce the strenght to zero, effectively nullifying it. Then, I painted along the crease in the mesh where the leg hits the pelvis to smooth out the area and remove the unsightly crease. This is a pretty interesting tool and I think that it will be very useful when working with un-weight-painted meshes.

Section 6 - 3: Animating Overlap Quickly

11-27-2023

As the title states, this was pretty quick. We animated a wagging dog tail. The tail is made up of three joints and as the base of the tail moves, the ones further down follow. All that was required to move the tail was to rotate the base of the tail by 20 degrees, the middle by 25, and the tip by 30. Made some start and end keyframes. Then using another handy dandy DASH EQUATION (mentioned in the last post) we multipled the value of each rotation by negative 1 to get the inverse position. Added another keyframe and we were met with a somewhat convincible wagging animation. However, it was still to rigid and blocky. To make the animation more fluid and make the weight of the tail appear as if it was moving like a real tail that follows the laws of physics, we went into the graph editor and moved the keyframes for the middle segment by two and the end segment by 4. This slight delay in the rotation of each joint allows the tail to move in a much more realistic way.





Section 6 - 2: Walk Cycle to Progressive Animation

11-27-2023

For this exercise, we started with a robot with a walk animation. The animation is composed of a single step with each foot. From there, we went into the animation graph editor and created a walk cycle by cycling the animation to infinity. Following this, we added the animation to an animation layer and then, on a separate layer, animated a translation on the Z axis for the moving components. To make the movement follow the proper speed of the walk cycle, we did something that I had never previously been taught or learned anywhere. It's this: look at the start position of one foot and then use the distance from that to the end position. After that, double it (one for each leg) and that is the proper value to add to each component to make it move at a constant rate that matches the animation. It is extremely simple, yet somehow never occured to me. I usually just try and eyeball it until it looks right so from now on I will be using this as a guideline for walk cycles. Ok, tangent aside, the movement was still kind of "slide-y" and the way to fix this is pretty common in all animation (I even use it for blender), in the graph editor, I just had to select the Z translation for each component and change the interpolation to linear. This removes any smoothing between frames and makes things move as they are meant to. Then it was just a matter of cycling the movement with the walk cycle and PRESTO! it moves.

One little aspect of this lesson that I found super helpful was learning how to use a function to add the same integer to each item's position without screwing everything up or having to do any math. For example, if we are translating each piece of the robot by 11 for each cycle of 25 frames, we can just select all of the geometry and type (in the specific value in the channel box) +=11 and it will add 11 to each of the existing integers that correspond with that item.

Section 4 - 3: Picking Up Objects

11-22-2023

In this exercise, we took an animation of a character moving his hand to a cup and bringing it up to his mouth and made the geometry of the cup follow his hand as he picked it up using parent constraints. By constraing the object at the frame in which it is picked up and then changing the value to zero on the keyframe before, the object will stay in place and then move with the parent once it reaches the desired frame. Another way of accomplishing this is to create a copy of the object, parent it to the character, and toggle the visibility of both objects on the frame where it is interacted with. The static object is visible until it reaches the interaction frame, and then the parented object becomes visible.





Section 4 - 2: Scaling Rigs

11-22-2023

In order to scale all of the contents of a rig without deformation of the geometry, skeleton, etc; The best way is to create something simple such as a nurbs circle and then parenting all of the necessary contents (except for the geometry) to it. If the geometry is also added then it will not scale at the same rate as the skeleton. Now, when scaling or moving the nurbs circle, it will also affect all of the parented objects.





Section 4 - 1: Using the Pose Editor in Skinning

11-22-2023

In this exercise, the pose editor was used to make a bicep "flex" and look less deformed when in a bent position. Essentially, the geometry is edited during one frame and the pose editor allows the object to morph between the two instances as it approaches that keyframe. So, when the arm is straight it is normal and when bent it has the large gaps filled. Using the pose editor, it is possible to change the geometry of an object in frames where it is necessary. By using the pose editor rather than adjusting the geometry on the fly, it is then possible to return the geometry to its original shape when not in a specific pose.





Section 3 - 4: Matrices Instead of Constraint in Rigs

11-14-2023

This exercise focused on using the Offset Parent Matrices from earlier in the course, this time in conjunction with a node in the Node Editor called pickMatrix. By running the matrix through a pickMatrix node, it allows for the separation of object transformation from the matrix. For instance, the Offset Parent Matrix lets the surfboard's translation, rotation, and scale alter that of the character that is being parented. By using the pickMatrix node and then deselecting "Use Scale" the surfboard can be scaled independently of the character. Another topic discussed in this video was the use of constraints which, in prior versions of Maya, was a more common way of parenting an object to another. By using a parent constraint, the transformation nodes of the child are unable to be edited as they are being controlled by the parent. However, using matrices, this is no longer an issue. The character can move completely independently from the parented surfboard without altering the surfboard or the matrix.

First LinkedIn Learning Course Finished

10-15-2023

Tonight I finished the Maya 2024 Essential Training course on LinkedIn Learning. It was a very good introduction to Maya and I definitely feel more comfortable using the software than before. The course consisted of 11 chapters and touched on the majority of the basic tools, menus, and techniques for modeling and making simple animations.

I am planning on using my notes and doing a writeup of everything the chapters went through and things that I find are implemented in a way that I feel is better or worse than in Blender. The course videos were brief with the majority of them being less than or equal to five minutes. I found myself needing to go back and re-watch the videos as I was following along with the project files.

I thought that the use of the project files throughout the course was useful yet also could have been done better. For instance, I would say that the best way to get someone to learn a program, especially one as dense with features as Maya, is to get them to iterate on top of the same project until it incorporates everything in each/every chapter. I am planning on doing this myself by making my own version of the course project from start to finish, rather than using one tool and then changing to the next file as was done in the course. Doing this would allow me to get further acquainted with Maya and have something to show that is actually my own work rather than a bunch of premade files in which I have only contributed with minor adjustments.
