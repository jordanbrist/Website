Section 4 - 3: Picking Up Objects

11-22-2023

In this exercise, we took an animation of a character moving his hand to a cup and bringing it up to his mouth and made the geometry of the cup follow his hand as he picked it up using parent constraints. By constraing the object at the frame in which it is picked up and then changing the value to zero on the keyframe before, the object will stay in place and then move with the parent once it reaches the desired frame. Another way of accomplishing this is to create a copy of the object, parent it to the character, and toggle the visibility of both objects on the frame where it is interacted with. The static object is visible until it reaches the interaction frame, and then the parented object becomes visible.





Section 4 - 2: Scaling Rigs

11-22-2023

In order to scale all of the contents of a rig without deformation of the geometry, skeleton, etc; The best way is to create something simple such as a nurbs circle and then parenting all of the necessary contents (except for the geometry) to it. If the geometry is also added then it will not scale at the same rate as the skeleton. Now, when scaling or moving the nurbs circle, it will also affect all of the parented objects.





Section 4 - 1: Using the Pose Editor in Skinning

11-22-2023

In this exercise, the pose editor was used to make a bicep "flex" and look less deformed when in a bent position. Essentially, the geometry is edited during one frame and the pose editor allows the object to morph between the two instances as it approaches that keyframe. So, when the arm is straight it is normal and when bent it has the large gaps filled. Using the pose editor, it is possible to change the geometry of an object in frames where it is necessary. By using the pose editor rather than adjusting the geometry on the fly, it is then possible to return the geometry to its original shape when not in a specific pose.





Section 3 - 4: Matrices Instead of Constraint in Rigs

11-14-2023

This exercise focused on using the Offset Parent Matrices from earlier in the course, this time in conjunction with a node in the Node Editor called pickMatrix. By running the matrix through a pickMatrix node, it allows for the separation of object transformation from the matrix. For instance, the Offset Parent Matrix lets the surfboard's translation, rotation, and scale alter that of the character that is being parented. By using the pickMatrix node and then deselecting "Use Scale" the surfboard can be scaled independently of the character. Another topic discussed in this video was the use of constraints which, in prior versions of Maya, was a more common way of parenting an object to another. By using a parent constraint, the transformation nodes of the child are unable to be edited as they are being controlled by the parent. However, using matrices, this is no longer an issue. The character can move completely independently from the parented surfboard without altering the surfboard or the matrix.





Section 3 - 3: Rigs for Dilating Pupils

11-14-2023

The focus on this exercise was using driven keys and blend shapes to dilate the pupil of an eyeball. Using a slider with driven keys causing the Y translation of the slider to blend the geometry as it moves from -1 to 1. At -1, the geometry is set to the narrow eye model, at 0 it is the regular eye model, and at 1 it becomes the dilated eye model. The blend shape allows the geometry to shift seemlessly between sizes and the driven keys make the change happen based on the location of the slider.

Section 2 - 4: Rivet Command

11-9-2023

By constraining an object to another using the rivet command, you can add an object to a scene that is already animated with a very short process. In the exercise, we riveted a hat model to a point on the mesh of a character that is walking and jumping. The hat stays in place with the geometry without deforming.





Section 2 - 3: Proximity Wrapping

11-9-2023

Proximity wrapping is used when conforming a mesh to another. In this exercise, we used a low resolution arm mesh with animation attached to it and proximity wrapped a higher resolution mesh (and a bracelet) to it. By doing this, one can animate a lower resolution mesh and cut down on processing power due to the reduced number of vertices. Then, when nearing the end of the project, a more detailed and high resolution mesh can be proximity wrapped and be used for the final render.

Section 1 - 4: Animation Timeline Bookmarks

11-1-2023

By setting bookmarks for portions of animations along the animation timeline, it becomes easier to find animated movements for duplication and editing. You can also zoom in on a specific bookmark to see only the transformations taking place in that portion of the timeline. This is great for organization and saving time later on when animating in chunks.

First LinkedIn Learning Course Finished

10-15-2023

Tonight I finished the Maya 2024 Essential Training course on LinkedIn Learning. It was a very good introduction to Maya and I definitely feel more comfortable using the software than before. The course consisted of 11 chapters and touched on the majority of the basic tools, menus, and techniques for modeling and making simple animations.

I thought that the use of the project files throughout the course was useful yet also could have been done better. I am planning on doing this myself by making my own version of the course project from start to finish, rather than using one tool and then changing to the next file as was done in the course. Doing this would allow me to get further acquainted with Maya and have something to show that is actually my own work rather than a bunch of premade files in which I have only contributed with minor adjustments.
